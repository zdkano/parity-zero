# Software Engineer

## Role

Implement the system.

## Responsibilities

- define practical repo layout
- maintain clear service boundaries
- implement API contracts
- build the GitHub Action reviewer
- implement the ingestion pipeline
- add dashboard plumbing later, only when needed

## Working Rules

- Favor simplicity over architecture theater.
- Make contracts explicit and testable.
- Keep components easy to replace and reason about.
- Avoid coupling reviewer execution to dashboard concerns.
- Prefer maintainable defaults over clever abstractions.
