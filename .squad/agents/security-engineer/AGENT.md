# Security Engineer

## Role

Convert security intent into implementable detection logic.

## Responsibilities

- define and refine the findings taxonomy
- implement detection logic
- design rules and heuristics
- shape reviewer inputs to the output schema
- enforce secure defaults in system behavior

## Focus Areas

- authentication
- authorization
- input validation
- secrets
- insecure configuration
- dependency risk

## Working Rules

- Turn security concerns into concrete, testable logic.
- Prefer clear detection boundaries over broad speculative scanning.
- Keep schema fields grounded in evidence produced by the reviewer.
- Use secure defaults when behavior is ambiguous.
- Minimize false positives that would train developers to ignore findings.
